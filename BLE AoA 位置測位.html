以下は、BLEのAoA測位においてIQサンプルから到来角（AoA）を求める一般的な手順の一例です。シリコンラボのAN1297にも、以下の考え方が示されています（​）。

1. IQサンプルから位相・振幅を求める

各アンテナから取得したI,Q値は、以下の関係式で表せます。

  I = A · cos(φ)
  Q = –A · sin(φ)

ここから、瞬時位相φは
  φ = atan2(–Q, I)
と求められ、振幅は
  A = √(I² + Q²)
で計算できます。（AN1297 3.3節）​

2. 位相補償

実際のCTE信号では、アンテナ切替によるサンプリング時刻のずれが存在します。

  • 参照期間の利用：最初の8サンプル（参照期間）では、すべて同一アンテナで取得しているため、ここから1μsあたりの実際の位相進み（通常は約90°前後、実測値により変動）を算出します。（AN1297 3.4節）​
  • 補正処理：その後のサンプル（スイッチング後の各サンプル）には、時間シフト分の位相進みが加わっているため、参照期間で求めた1μs当たりの位相シフトを用いて各サンプルの位相を補正します。スロット長が1μsなら補正係数は2、2μsなら4倍する必要がある点にも注意してください。

3. アンテナアレイのジオメトリと平面波モデルの適用

4×4の均一矩形アレイ（各アンテナの位置は、仕様書で示されているように既知）では、平面波が到来した場合、各アンテナでの理論上の位相は以下のようにモデル化できます。

  φ(x, y) = φ₀ + (2π/λ) · [ x · sin(θ) cos(ϕ) + y · sin(θ) sin(ϕ) ]

  • λ：信号の波長（例：2.4GHzなら約125mm）
  • (x, y)：各アンテナの相対座標（アレイ中心を基準とする）
  • θ：仰角（水平面との角度）
  • ϕ：方位角（例えば、仕様書では上側を180°、右側を90°などと定義）

各アンテナで補正済みの実測位相と、このモデルによる予測位相との差が最小になるように、θやϕ（もしくは1次元の場合は単一角度）を推定します。具体的には、最小二乗法やグリッドサーチ、あるいはMUSICなどの高精度アルゴリズムを利用して、最も適合する入射方向を決定します。

4. 角度から位置推定へ

単一のAoA推定ではタグの方向は得られますが、正確な位置測定（2次元または3次元の座標）には、複数箇所での測定または複数のアンテナシステムによる三角測量（トライラテレーション）が必要です。

  • 例えば、複数の基地局からのAoAデータを組み合わせることで、タグの位置を求めることが可能になります。

まとめ

  1. IQ変換：各アンテナのI,Q値から φ = atan2(–Q, I) を計算
  2. 位相補償：参照期間のサンプルから1μsあたりの位相進みを求め、各サンプルの位相に補正を適用
  3. モデル適用：各アンテナの補正済み位相と、アレイの既知の位置情報を用いて、平面波モデル（φ(x,y) = φ₀ + (2π/λ)[x sin(θ) cos(ϕ) + y sin(θ) sin(ϕ)]）を構築
  4. 角度推定：実測位相と理論値の差を最小化する方法で、入射角（θ, ϕまたは単一角度）を求める
  5. 位置決定：複数のAoA推定値を用いて、タグの位置を三角測量により算出

以上の計算方法により、4×4アンテナアレイのIQサンプルからBLEタグの到来方向を求め、複数観測点を組み合わせることで位置測位が実現されます。

このような手法は、シリコンラボのAN1297に詳細が記載されており、実装にあたっては補正処理やノイズ対策、アルゴリズム選定などの検討が必要となります。
